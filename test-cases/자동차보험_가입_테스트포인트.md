# 자동차보험 가입 프로세스 테스트 포인트

## 개요

 - 본 문서는 국내 다이렉트 자동차보험 가입 프로세스를 기준으로 주요 테스트 포인트를 정리한 학습용 포트폴리오 문서이다.
 - 특정 보험사나 특정 시스템의 상세 구현 방식은 제외하고,공통적인 가입 흐름과 실제 보험 실무 경험에서 자주 확인했던 업무 규칙 및 요구사항 기반 검증 포인트를 중심으로 작성하였다.
 - 본 문서는 기능 개선이나 정책 변경을 제안하기 위한 목적이 아니며, 보험 업무 규칙을 기준으로 검증이 필요한 항목을 정리하는 것을 목적으로 한다.
 - 모니터링 로그는 운영 중 상담 및 오류 원인 확인을 위해 사용하는 운영 화면의 조회 정보를 의미한다.

## 공통 검증 항목

- 단계 이동(이전/다음) 시 입력 정보 유지 여부 검증
- 세션 유지 여부 검증
- 브라우저 뒤로가기 시 화면 및 입력 데이터 처리 검증
- 브라우저 새로고침 처리 검증

## 1-1. 개인정보 입력

### Test Point

- 필수 입력값 누락 검증
- 입력값 형식 검증
- 가입설계동의 체크 여부 검증
- 실명인증 실패 처리 여부 검증
- 보험 가입 가능 연령에 따른 가입 제한 검증

#### Customer Note (실제 업무 사례)

1. 보험나이 기준 변경 시점(연초)에는 가입 가능 여부에 대한 문의가 자주 발생하였다. 

가입 가능 연령 기준이 요구사항대로 적용되는지 함께 확인하였다.

2. 실명인증 실패 및 개인정보 입력 오류로 문의가 자주 발생하였다. 

오류 메시지와 입력 제한이 요구사항대로 동작하는지 함께 확인하였다.

## 1-2. 본인인증

### Test Point

- 보험정보망 조회 동의 체크 여부 검증
- 인증 성공 및 실패 처리 여부 검증 (조회 결과가 화면에 정상 반영되는지 검증)
- 인증번호 발송 여부 검증
- 인증 실패 처리 여부  검증
- 카드 인증 실패 처리 검증

#### Customer Note (실제 업무 사례)

1. 카드 인증 시 키보드 보안 프로그램으로 인해 카드번호 입력이 정상적으로 되지 않는 문의가 자주 발생하였다.

보안 프로그램 설치 상태에 따른 입력 가능 여부를 함께 확인하였다.

2. 카드 비밀번호 입력 오류로 카드 인증이 제한되는 경우에 대한 문의가 자주 발생하였다.

인증 실패 횟수에 따른 제한 처리가 요구사항대로 동작하는지 함께 확인하였다.

## 2-1 자동차 정보 및 보험기간

### Test Point

- 차종 선택 필수값 검증
- 차량번호 및 차대번호 형식 검증
- 보험기간 입력 검증
- 동일 차량 중복가입 검증- 차량 구입 예정 설계와 차량번호 입력 설계 간 보험료 일관성 검증
- 보험기간별 마일리지 특약 가입 가능 여부 검증
- 보험기간 및 차종별 동일증권 선택 가능 여부 검증
> **동일증권**: 보험만기를 맞춰 하나의 증권으로 관리하는 가입 방식
- 보험기간별 할인특약 가입 가능 여부 검증

### 업무규칙 참고사항

① 보험기간이 3개월 이상인 경우 마일리지 특약 가입이 가능하다.
② 보험기간, 차종 및 보험만기일 조건을 만족하는 경우 동일증권 선택이 가능하다.
③ 보험기간이 3개월 이상인 경우 할인특약 가입이 가능하다.
④ 보험기간이 1년 미만인 경우 단기요율이 적용되고, 1년인 경우 일할 계산 기준이 적용된다.

#### Customer Note (실제 업무 사례)

계약자 변경을 위해 동일 차량으로 신규 가입을 진행하는 과정에서 동일 차량 중복가입으로 가입이 제한되는 문의가 자주 발생하였다.

구입 예정 차량으로 설계하는 경우에는 가입 설계가 가능하므로, 업무규칙에 맞게 중복가입 제어 여부를 함께 확인하였다.

## 2-2. 자동차 상세 정보 입력

### Test Point

#### 입력 및 선택
- 추가 부속 선택 시 필수 입력값 누락 검증
- 차종에 따른 할인부속 선택 가능 여부 검증
#### 화면 연동
- 할인부속 직접 입력 선택 시 사진 등록 화면 활성화 여부 검증
- 선택된 차종 변경 시 기존 부속 정보 초기화 검증
#### 보험료 반영
- 할인부속 장착 여부에 따른 보험료 반영 검증
- 외부 차량정보 조회 성공 시 할인부속 반영 검증
- 외부 차량정보 조회 실패 시 예외 처리 검증
#### 중복 및 데이터 처리
- 외부 차량정보 조회 결과와 고객 직접 입력 할인부속 중복 제어 검증
- 자동 조회된 할인부속 삭제 처리 검증
#### 갱신
- 갱신 가입 시 기존 할인부속 정보 반영 및 보험료 계산 검증
- 신규 할인부속 적용 시 외부 차량정보 조회 정보 기반 보험료 계산 검증
#### 예외
- 외부 차량정보 부속 정보 및 직접입력한 부속정보 모니터링 로그 적재 여부 검증

### 업무규칙 참고

① 커넥티드카 정보가 존재하는 경우 관련 특약이 자동 적용된다.
② 외부 차량정보 조회 결과가 존재하는 경우 할인부속 정보가 자동 반영되며, 기존 직접 입력 정보와 중복되지 않도록 제어한다.

#### Customer Note (실제 업무 사례)

갱신 가입 시 기존 직접 입력한 할인부속과 외부 차량정보 조회된 할인부속이 중복 등록되어 가입이 중단되는 사례가 있었다.

자동 조회 정보와 고객 입력 정보가 중복되지 않도록 업무규칙에 맞게 제어되는지 함께 확인하였다.

## 3-1. 운전자정보 

### Test Point

- 운전자 정보 필수 입력값 누락 여부 검증
- 최저연령자 기준 연령선반영 문구 노출 검증
- 운전범위별 할인 특약 화면 노출 여부 검증
- 갱신 가입 시 기존 운전범위가 자동 적용되는지 검증

### 업무규칙 참고

① 최저연령자 기준에 따라 연령선반영 문구가 자동 적용된다.
② 자녀운전자 한정 선택 시 기명피보험자 운전 불가 문구가 자동 노출된다.

#### Customer Note (실제 업무 사례)

갱신 가입 시 최저연령자의 연령선반영 문구를 인지하지 못해 문의하는 사례가 자주 발생하였다.

신규 및 갱신 가입 시 연령선반영 문구가 요구사항에 맞게 노출되는지 함께 확인하였다.

## 3-2. 할인 특약

### Test Point

#### 가입
- 할인특약 가입 시 필수 입력값 검증
#### 조회
- 할인특약 조회 결과 반영 검증
#### 보험료
- 할인특약 신규 가입 후 보험료 재계산 검증
- 할인특약 해제 후 보험료 재계산 검증
- 할인특약 정보 변경 시 보험료 재계산 검증
#### 예외
- 할인특약 조회 결과 오류시 예외 처리 검증
- 조회 시점에 따른 할인특약 적용 검증
- 조회 결과가 모니터링 로그에 정상적으로 적재되는지 검증
#### 갱신
- 갱신 가입 시 기존 할인특약 정보 반영 검증

### 업무규칙 참고

 ① 안전운전 할인특약끼리 중복가입이 불가하다. 
 ② 당사 자녀보험 가입 고객은 자녀 할인특약이 자동 적용된다.
 ③ 동일 할인특약 중복가입 불가하다.
 ④ 조회 결과가 변경되더라도 가입 가능한 정보가 존재하는 경우 할인특약이 자동 적용된다.

#### Customer Note (실제 업무 사례)

조회 결과에 따라 할인율이 변경되어 보험료가 달라지는 경우 문의가 자주 발생하였다.

조회 결과가 변경되는 특약은 구간별 할인율과 보험료 반영 여부를 함께 확인하였다.

## 4. 보험료 계산

### Test Point

- 담보 추가 시 보험료 재계산 검증
- 담보 삭제 시 보험료 재계산 검증
- 특약 추가 및 해제 시 보험료 재계산 검증
- 보험료 계산 결과와 최종 결제 금액 일치 여부 검증
- 보험료 계산 실패 시 오류 처리 검증

#### Customer Note (실제 업무 사례)

전기차의 경우 연계담보 자동가입과 신설 할인특약이 함께 적용되면서 보험료 변경 문의가 자주 발생하였다.

차종별 할인특약과 연계담보가 업무규칙에 맞게 적용되어 보험료가 정상적으로 계산되는지 함께 확인하였다.

## 5-1. 피보험자 / 계약자 정보

### Test Point

- 계약자와 피보험자가 다른 경우 입력 화면 노출 검증
- 계약자/피보험자의 필수 입력값 누락 여부 검증
- 질권 설정 체크 시 리스 사업자 정보가 정상적으로 불러오는지 검증
- 계약자 정보에 피보험자 이름, 주민번호 입력 방지 검증
- 질권 정보 입력 시 정상적으로 모니터링 로그에 적재되는지 검증

#### Customer Note (실제 업무 사례)

갱신 가입 시 기존 계약자가 아닌 다른 계약자로 가입을 진행하는 방법에 대한 문의가 자주 발생하였다.

계약자 변경 시 입력 화면과 가입 절차가 요구사항에 맞게 동작하는지 함께 확인하였다.

## 5-2. 사진 등록

### Test Point

- 필수 사진 등록 누락 여부 검증
- 동일사진 등록 방지 여부 검증
- 사진등록 후 이탈 후 재진입시 등록한 사진 유지 여부 검증
- 갱신가입시 기존 등록한 사진 유지 여부 검증
- 허용되지 않는 파일 형식 업로드 제한 검증
- 파일 용량 초과 시 예외 처리 검증

#### Customer Note (실제 업무 사례)

전년도 커넥티드 마일리지 특약으로 가입한 고객이 갱신 시 커넥티드 정보가 조회되지 않아 사진형 마일리지 특약으로 전환되는 사례가 있었다.

조회 결과에 따라 사진 등록 화면이 요구사항에 맞게 정상적으로 노출되는지 함께 확인하였다.

## 6. 청약 내용 확인

### Test Point

- 필수 체크 표시 누락 여부 검증
- 입력한 가입 정보가 청약 내용에 정확하게 반영되는지 검증
- 가입설계서, 보험약관, 상품설명서 선택시 전자문서 뷰어 활성화 검증

#### Customer Note (실제 업무 사례)

신설 특약 적용 시 청약 내용 화면에 특약명이 NULL로 표시되는 사례가 있었다.

신설 특약 적용 여부에 따라 청약 내용이 요구사항대로 정상 표시되는지 함께 확인하였다.

## 7. 보험료 결제

### Test Point

- 결제 정보 입력값 누락 여부 검증
- 페이 결제 시 결제화면이 정상적으로 활성화되는지 검증
- 결제 실패 시 가입 완료 처리 여부 검증
- 결제 실패시 입력한 결제정보 화면 내 유지 검증
- 결제 실패시 정상적으로 모니터링 로그에 적재되는지 검증
- 계약체결동의 체크 여부 검증
- 중복 결제 방지 검증
- 무통장 입금 후 카드 결제 전환 검증
- 무통장 입금 후 실시간이체 전환 검증
- 결제 완료 후 중복 클릭 방지 검증

#### Customer Note (실제 업무 사례)

페이 결제 진행 중 비정상 종료로 인해 보험료 결제 화면으로 복귀하지 못하는 사례가 있었다.

비정상 결제 종료 후 보험료 결제 화면으로 정상 이동되는지 함께 확인하였다.

## 8. 가입 완료

### Test Point

- 가입 완료 후 카카오알림톡으로 가입 완료 안내 및 증권·약관 발송 검증
- 무통장 입금 시 가상계좌 번호 발송 검증
- 가입 완료 후 계약 조회 가능 여부 검증

### 업무규칙 참고

카카오알림톡 수신이 불가능한 경우 문자로 안내정보가 발송된다.

#### Customer Note (실제 업무 사례)

무통장 입금 완료 후 증권 및 약관을 받지 못했다는 문의가 자주 발생하였다.
