# 자동차보험 가입 프로세스 테스트 분석 및 테스트 포인트 도출

## 개요

본 문서는 국내 다이렉트 자동차보험 서비스의 일반적인 가입 절차를 기준으로 테스트 포인트를 도출하였다.
특정 보험사의 구현 방식이나 내부 자료가 아닌 공통 가입 흐름을 기준으로 하며, 보험 고객·운영 및 서비스 변경 검증 경험을 바탕으로 개인이 재구성하였다.

## 공통 검증 항목

- 이전 단계 복귀 시 입력 정보 유지 여부
- 현재 선택값과 숨겨진 이전값의 분리 여부
- 세션 유지 및 만료 처리
- 브라우저 뒤로가기·새로고침 처리
- 변경된 입력값이 다음 단계에 동일하게 전달되는지 확인
- 오류 발생 시 이전 결과 잔존 여부와 재시도 가능 여부

## 1. 가입자 확인

### 1-1. 개인정보 입력

- 필수 입력값 누락 검증
- 입력값 형식 검증
- 가입설계동의 체크 여부 검증
- 실명인증 실패 처리 여부 검증
- 입력제한 길이 검증
- 특수문자 입력 여부 검증

### 1-2. 본인인증

- 보험정보망 조회 동의 체크 여부 검증
- 인증수단별 인증 성공·실패 상태 구분
- 인증번호 발송 여부 검증
- 인증 실패 시 오류 안내와 재시도 여부 검증
- 인증 완료 전 다음 단계 진행 제한 여부 검증

## 2. 자동차 정보

### 2-1. 자동차 정보 및 보험기간

- 차량번호·차대번호 형식 검증
- 필수 선택값 누락 검증
- 보험기간 입력 검증
- 동일 차량의 중복 가입 여부 검증
- 차량 구입 예정으로 설계할 때 할인부속과 할인특약이 보험료에 정상 반영되는지 검증

### 2-2. 자동차 상세 정보 입력

- 추가 부속 선택 시 필수 입력값 누락 검증
- 할인부속 직접 입력 선택 시 사진 등록 화면 활성화 여부 검증
- 차량 변경 후 이전 차량의 할인부속·추가부속 정보 잔존 여부 검증
- 자동조회 부속과 사용자가 입력한 추가부속의 중복 적용 여부 검증
- 부속가액이 차량가액과 보험료에 중복 합산되지 않는지 검증

## 3. 운전자 범위 및 할인특약

### 3-1. 운전자 범위

운전자 범위는 실제 운전 가능한 사람을 제한하는 조건이며, 운전자 정보·가입 가능한 할인특약·보험료 계산 결과에 영향을 준다.
따라서 선택 항목의 노출만 확인하지 않고, 범위 변경 후 현재 선택값에 필요한 정보만 보험료와 청약정보에 사용되는지 확인한다.

#### 운전자 범위 선택값

계약 조건과 상품에 따라 다음 항목이 노출될 수 있다.

- 기명피보험자 한정
- 부부 한정
- 가족 한정
- 기명피보험자 1인 및 지정운전자 1인
- 누구나 운전
- 임직원 한정
- 자녀 한정

#### 주요 업무 규칙

| 구분 | 검증 규칙 |
| --- | --- |
| 신규가입 | 입력한 운전자 정보가 유지되더라도 최종 선택한 운전자 범위에 필요한 값만 계산에 사용되어야 한다. |
| 갱신가입 | 전년도 운전자 정보가 불러와진 경우 현재 계약의 운전자 범위와 연령 조건에 맞는지 다시 확인해야 한다. |
| 지정운전자 | 다른 범위로 변경하면 지정운전자 입력영역은 숨겨질 수 있으며, 숨겨진 값이 현재 보험료와 청약정보에 사용되면 안 된다. |
| 범위 재선택 | 기명피보험자 1인 및 지정운전자 1인으로 다시 변경하면 기존 지정운전자 정보가 복원될 수 있다. |
| 최저연령 | 현재 날짜가 아니라 보험 계약 개시일을 기준으로 판단한다. |
| 연령선 반영 | 보험기간 중 연령 조건이 변경되는 경우 적용 예정일이 화면 안내 문구와 일치해야 한다. |
| 할인특약 | 운전자 범위에 따라 가입 가능한 할인특약이 달라질 수 있다. |
| 보험료 확인 | 운전자 범위 변경 결과는 보험료 계산결과 단계에서 최종 확인한다. |

#### 상태 변경 검증

| 이전 상태 | 사용자 행동 | 기대 결과 | 주요 위험 |
| --- | --- | --- | --- |
| 지정운전자 정보 입력 완료 | 다른 운전자 범위로 변경 | 지정운전자 영역은 숨겨지고 해당 값은 현재 계산에서 제외 | 숨겨진 지정운전자 정보가 보험료·청약정보에 사용됨 |
| 다른 운전자 범위 선택 | 지정운전자 범위로 재변경 | 기존 지정운전자 정보가 일관되게 복원 | 이름·생년월일·관계값 중 일부만 복원 |
| 할인특약 선택 완료 | 호환되지 않는 운전자 범위로 변경 | 할인특약 해제 또는 비활성화와 변경 사유 안내 | 화면에서만 사라지고 할인보험료는 유지 |
| 보험료 결과 확인 완료 | 이전 화면에서 운전자 범위 변경 | 보험료 결과 재진입 시 변경된 범위로 재계산 | 이전 운전자 범위의 보험료 잔존 |
| 갱신가입 진입 | 전년도 운전자 정보 불러오기 | 현재 선택값과 개시일 기준으로 다시 검증 | 전년도 정보가 확인 없이 적용 |

#### 핵심 검증 위험

- `DRV-01`: 숨겨진 지정운전자 정보가 현재 보험료 또는 청약정보에 사용됨
- `DRV-02`: 운전자 범위 변경 후 가입 불가능한 할인특약이 보험료에 남음
- `DRV-03`: 운전자 범위 변경 후 이전 보험료가 결과 화면에 잔존함
- `DRV-04`: 갱신가입 시 전년도 운전자 정보가 현재 계약에 그대로 적용됨
- `DRV-05`: 할인특약 화면이 사라진 이유가 안내되지 않아 사용자가 오류로 인식함

### 3-2. 할인특약

- 운전자 범위와 차량·운전자 조건에 맞는 할인특약만 노출되는지 검증
- 할인특약 가입 시 필수 입력값 누락 여부 검증
- 할인특약 선택·해제 후 보험료가 정상적으로 재계산되는지 검증
- 할인특약 정보 변경 시 적용 개시일과 구간별 할인율이 정확한지 검증
- 할인특약 가입 여부와 보험료 결과 화면의 할인내역이 일치하는지 검증
- 조회 실패와 가입 대상 아님 상태가 명확히 구분되는지 검증
- 운전자 범위 변경으로 할인특약이 제외될 때 기존 할인금액이 보험료에 남지 않는지 검증

## 4. 보험료 계산결과

### 4-1. 화면 표시정보

- 담보명
- 담보 가입금액 또는 보상한도
- 담보별 보험료
- 최종보험료
- 주행거리 할인 적용 전·후 보험료
- 할인특약 가입 여부
- 할인내역 상세
- 전년도 계약과의 비교정보

주행거리 할인 전·후 보험료나 전년도 계약 비교 기능은 상품과 서비스에 따라 다를 수 있으므로 해당 기능이 제공되는 경우에 검증한다.

### 4-2. 보험료 재계산 규칙

| 변경 위치 | 사용자 행동 | 기대 결과 |
| --- | --- | --- |
| 이전 단계 | 차량·운전자 범위·연령·할인특약 변경 | 보험료 계산결과 재진입 시 최종 변경값으로 자동 계산 |
| 보험료 결과 화면 | 담보 또는 가입금액 변경 | 변경값을 기준으로 담보별 보험료와 최종보험료 갱신 |
| 별도 저장 방식 | 담보 변경 후 변경 내용 저장 | 저장 완료 후 최신 보험료 표시 |
| 보험료 결과 확인 후 이전 단계 복귀 | 계약 조건 변경 후 다시 진입 | 이전 보험료가 아닌 변경된 조건의 보험료 표시 |
| 결제 단계 이동 | 보험료 결제 화면 진입 | 보험료 계산결과의 최종보험료와 결제금액 일치 |

서비스에 따라 담보 변경과 동시에 보험료가 재계산되거나 별도 저장 동작이 필요할 수 있다.
따라서 특정 화면 방식보다 최종 변경값이 보험료 계산과 결제에 동일하게 사용됐는지를 확인한다.

### 4-3. 계산 상태 및 오류 처리

| 상태 | 확인할 화면 동작 | 검증 내용 |
| --- | --- | --- |
| 계산 완료 | 담보별 보험료와 최종보험료 표시 | 최신 계약 조건과 계산 결과 일치 |
| 조건 변경 후 미저장 | 기존 보험료가 표시될 수 있음 | 과거 보험료를 최종 금액으로 오인하지 않도록 상태 구분 |
| 재계산 중 | 로딩 화면 표시 | 중복 계산 요청과 화면 이탈 처리 |
| 계산 실패 | 로딩 중 멈춤·빈 화면·현재 단계 잔존 | 명확한 오류 안내, 이전 결과 무효화, 재시도 가능 여부 |
| 결제 단계 전달 | 결제금액 표시 | 보험료 계산결과 및 기간계 보험료와 일치 |

### 4-4. 보험료 정합성 비교

보험료 또는 할인율 변경 테스트에서는 다음 항목을 함께 비교한다.

1. 변경 대상 담보·특약의 보험료
2. 최종보험료
3. 특약 적용 개시일 전·후 보험료
4. 기간계·웹·모바일 보험료
5. 보험료 계산결과와 결제단계 금액
6. 결제 후 가입 완료 상태

#### 핵심 검증 위험

- `PRM-01`: 담보 변경 후 저장되지 않아 이전 보험료가 결과 화면에 남음
- `PRM-02`: 보험료 계산결과와 결제금액이 다름
- `PRM-03`: 기간계·웹·모바일의 보험료가 다름
- `PRM-04`: 할인율 적용 개시일 전·후에 잘못된 요율이 적용됨
- `PRM-05`: 계산 실패 시 로딩 또는 빈 화면만 남아 재시도할 수 없음
- `PRM-06`: 할인특약 가입 여부와 실제 할인보험료가 일치하지 않음
- `PRM-07`: 이전 단계 변경값이 반영되지 않은 보험료가 결제단계로 전달됨

### 4-5. 운영 문의 기반 검증 사례

고객센터 문의를 바탕으로 담보를 여러 차례 변경하는 가입 흐름을 테스트계와 운영계에서 재현하였다.
보험료 계산결과와 결제단계 금액이 일치하지 않아 기간계 보험료까지 비교한 결과, 결제단계 금액은 기간계와 일치했지만 보험료 계산결과 화면에는 최종 변경사항이 반영되지 않은 금액이 표시되고 있었다.
확인 내용을 현업과 개발 담당자에게 전달했으며, 수정 후 보험료 계산결과·결제금액·기간계 보험료가 일치하는지 재검증하였다.

### 4-6. 운영 이관 후 회귀 범위

보험료 관련 변경사항은 웹·모바일·기간계를 함께 비교하고, 보험료 계산 이후 결제와 가입 완료까지 확인한다.
운영 반영 후 중대한 오류가 발생해 현업이 롤백을 결정하거나 개발자가 긴급 수정한 경우에도 테스트계와 운영계에서 동일한 범위를 재검증한다.

완료 기준은 다음과 같다.

- 웹·모바일·기간계 보험료 일치
- 담보·특약 보험료와 최종보험료 일치
- 적용 개시일 전·후 보험료 정상 반영
- 보험료 계산결과와 결제금액 일치
- 결제 후 가입 완료 정상 처리
- 수정사항의 테스트계 및 운영계 반영 일치

## 5. 피보험자·계약자 정보 및 사진

### 5-1. 피보험자·계약자 정보

- 계약자와 피보험자가 다른 경우 계약자 정보 입력 화면 노출 여부 검증
- 계약자·피보험자의 필수 입력값 누락 여부 검증
- 질권설정 선택 시 리스 사업자 정보가 정상적으로 호출되는지 검증
- 계약자 정보에 피보험자 정보가 잘못 중복 입력되지 않는지 검증
- 질권 정보가 다음 단계와 청약정보에 동일하게 전달되는지 검증

### 5-2. 사진 등록

- 필수 사진 등록 누락 여부 검증
- 동일 사진 등록 방지 여부 검증
