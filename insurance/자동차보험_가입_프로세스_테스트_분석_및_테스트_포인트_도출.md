# 자동차보험 가입 프로세스 테스트 분석 및 테스트 포인트 도출

## 개요

본 문서는 국내 다이렉트 자동차보험 서비스의 일반적인 가입 절차를 기준으로 테스트 포인트를 도출하였다.
특정 보험사의 구현 방식이 아닌 공통적인 자동차보험 가입 프로세스를 기반으로 작성하였으며, 실제 보험 운영 경험을 바탕으로 주요 검증 포인트를 정리하였다.

## 공통 검증 항목

- 이전 단계 복귀 시 입력 정보 유지 여부
- 세션 유지 여부 
- 브라우저 뒤로가기 처리

1-1. 개인정보 입력
- 필수 입력값 누락 검증
- 입력값 형식 검증
- 가입설계동의 체크 여부 검증
- 실명인증 실패 처리 여부 검증
- 입력제한 길이 검증
- 특수문자 입력 여부 검증

1-2. 본인인증
- 보험정보망 조회 동의 체크 여부 검증
- 인증수단별 인증성공/실패시 로그가 정상적으로 적재되는지 검증
- 인증번호 발송 여부 검증
- 인증실패 처리 여부  검증

2-1. 자동차 정보 및 보험기간
- 차량/차대번호 형식 검증
- 필수 선택값 누락 검증
- 보험기간 입력 검증
- 동일 차량의 중복 가입 여부 검증
- 차량 구입 예정으로 설계 시 할인부속 및 할인특약이 반영된 보험료가 정상적으로 계산되는지 검증

2-2. 자동차 상세 정보 입력
- 추가 부속 선택 시 필수 입력값 누락 검증
- 할인 부속 직접 입력 선택 시 사진 등록 화면 활성화 여부 검증
- 자동 할인 부속 삭제 여부 검증

3-1. 운전자 정보
- 운전자 정보 필수 입력값 누락 여부 검증
- 최저 연령자 기준 연령 구간 별 연령 선 반영 문구 노출 검증
- 운전자 정보, 관계값이 정상적으로  로그에 적재되는지 검증
- 운전자 범위별  할인 특약 화면 노출 여부 검증

3-2. 할인 특약
- 할인 특약 가입시 API 호출 여부 검증
- 할인 특약 가입시 필수입력값 누락 여부 검증
- 할인 특약 가입시 보험료가 정상적으로 재계산되는지 검증
- 할인 특약 정보 변경시 구간별 할인율로 보험료가 정상적으로 재계산되는지 검증
- 할인 특약 조회시 API 정보가 로그에 정상적으로 적재되는지 검증
- 할인 특약 API 정보 오류시 보험료가 재계산되는지 검증

1. 보험료 계산
- 담보 및 특약 변경시 보험료가 정상적으로 재계산되는지 검증
- 설계한 정보로  견적받기 화면 내 문구 일치 여부 검증
- 보험료 계산 API 오류 처리 여부 검증
- 보험료 계산 결과가 화면과 최종 결제 금액에 동일하게 반영되는지 검증

5-1. 피보험자 / 계약자 정보
- 계약자/피보험자 상이 가입시 계약자 정보 입력 화면 노출 여부 검증
- 계약자/피보험자의 필수 입력값 누락 여부 검증
- 질권설정 체크시 리스 사업자 정보가 정상적으로 호출되는지 검증
- 계약자 정보에 피보험자 이름, 주민번호 입력 방지 검증
- 질권 정보 입력시 정상적으로 로그에 적재되는지 검증

5-2. 사진 등록
- 필수 사진 등록 누락 여부 검증
- 동일사진 등록 방지 여부 검증
- 사진등록 후 이탈후 재진입시 등록한 사진 유지 여부 검증

6. 청약 내용 확인
- 필수 체크 표시 누락 여부 검증
- 입력한 정보와 동일한 문구로 노출되는지 검증

7. 보험료 결제
- 결제 정보 입력값 누락 여부 검증
- 페이 결제시 결제화면이 정상적으로 호출되는지 검증
- 결제 실패시 가입 처리 여부 검증
- 결제 실패시 입력한 결제정보 화면 내 유지 검증
- 결제 실패시 정상적으로 로그가 적재되는지 검증
- 계약체결동의 체크 여부 검증
- 중복 결제 방지 검증
- 결제 중 새로고침 처리 여부 검증

8. 가입 완료
- 가입 완료 후 카카오알림톡 발송 검증
- 증권, 약관 발송 검증
- 카카오 수신거부시 문자 정상발송 검증
- 무통장 입금시 가상계좌 번호 발송 검증
